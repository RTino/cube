How to make your own Cube

Once you get the included example up and running, it will be easy to set up the cube to handle your own data. This small tutorial will show you how to store a collection of books in the cube. As you will see it's basically a matter of writing a schema, editing a couple of config files and setting up the database.

  1. Add a new Solr core

    Add a core tag in solr/solr/solr.xml file. At the end it should look something like this

    solr/solr/solr.xml<?xml version="1.0" encoding="UTF-8" ?>
    <solr sharedLib="lib" persistent="true">
      <cores adminPath="/admin/cores">
        <core name="team" instanceDir="conf/cube/" dataDir="conf/cube/team" />
        <core name="library" instanceDir="conf/cube/" dataDir="conf/cube/library" />
      </cores>
    </solr>
  2. Start Solr
    $ cd cube/solr/
    $ mvn clean install tomcat:run-war -Pdevelopment

    You should have a new core called "library" by now. This is where all your library collection will be stored. Check it out on http://localhost:38730/cube-solr.

  3. Create the entity's files
    $ cd util/
    $ ./create_entity library
  4. Configure
    1. Schema

      The Schema is one of the most important aspects of your entity. We recommend you to spend some time checking out the Schema Reference and the Team example schema.

      For this tutorial a schema is provided. Get it from here and place it in entities/library/

      $ cd cube/entities/library
      $ curl -O http://cubeapp.io/schema.json
    2. Database config

      Configure entities/library/db.json to point to the new Solr core

      It should look something like this:

      entities/library/db.json{
        "host": "localhost",
        "port": "38730",
        "core": "library",
        "path": "/cube-solr"
    }
    3. Entity Information

      We need to provide some information about the entity along with some settings. Please take a look at entities/library/settings.json. It should look something like this:

      entities/library/settings.json{
        "entity": "library",
        "title": "Library",
        "description": "My huge book collection",
        "itemType": [ "book", "books" ],
        "separator": "/",
        "view": "list",
        "rows": 50,
        "sort": "title:asc"
    }
  5. Add your entity to the list

    Open the file called entities.json and add it to the array

    entities.json[ "team", "library" ]
  6. Start NodeJs
    $ coffee cube.coffee
  7. Visit your library
    http://localhost:3000/library/?admin=yes

    * Currently there is no authentication support in the Cube App (its on its way!), so for now the way to be able to add/edit all data from any entity, is by appending the "admin=yes" parameter to the querystring.

  8. Add your books!

    Click Addon the top right corner and start adding your books!
