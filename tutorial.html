Adding a new extension

Once you got the included example up and running, it will be no hassle to set up your own extension. All you need to do is set up a new solr core and place your extension files inside the extensions directory. Follow these steps to get it done:

  1. Add a solr core

    Edit your solr/solr/solr.xml file. At the end it should look something like this

    <?xml version="1.0" encoding="UTF-8" ?>
<solr sharedLib="lib" persistent="true">
  <cores adminPath="/admin/cores">
    <core name="team" instanceDir="conf/cube/" />
    <core name="myExtension" instanceDir="conf/cube/" />
  </cores>
</solr>
  2. Start Solr

    * stop it if its running already, duh!

    $ cd solr/
    $ mvn clean install tomcat:run-war -Pdevelopment

    You should have a new core now. Check it on http://localhost:38730/cube-solr

  3. Add a new entity on entities.json

    entities.json is a json file! beware of the syntax. It should look like this:

    [ "team", "myExtension" ]
  4. Copy example directory with your desired extension name
    $ cp extensions/team extensions/myExtension
  5. Configure your extension

    1. Write your own Schema

      The Schema is one of the most important aspects of your extension. We recommend you to spend some time checking out the one provided in the example along with its documentation.

      Please check the Schema reference here.

    2. Configrue extensions/myExtension/db.json to point to your new Solr core

      It should look something like this:

      {
    "host": "localhost",
    "port": "38730",
    "core": "myExtension",
    "path": "/cube-solr"
}

    3. Edit the CSS to fit your style on extensions/myExtension/style.styl

  6. Start NodeJs
    $ coffee cube.coffee
