Documentation

The Cube is composed of a NodeJs backend, a Solr database and a BackboneJs frontend. If you're familiar with these technologies it shouldn't be much of a hassle to get around this app; For the developer in you, annotated source code documentation generated with docco can be found here.

Schema reference

One of the main tasks when creating your own entity is writing a Schema. A Cube Schema is a json array of objects that describes the structure of your data collection. You can take a look at an example schema which makes use of almost all possible properties. Setting up yours is a matter of listing the fields you want your items to have with their IDs, Labels and a custom set of properties like field type (img, email, facet, etc).

Available Properties
key type default description
id String Each field must have a unique ID. This field is required for Solr, but not for the Cube. If defined in the Schema, it will become visible in your Cube, otherwise it will only be used by the database.
label String IDs are not necessarily pretty or even human readable. A label is a must. Alphanumeric characters and spaces. Value type
type String String Specifies which type of data will the field contain. It tells the cube how to store this value and also how to display it. Default (no type property specified) is String. For a list of types, consult the types table bellow.
mandatory Boolean False Marks field to be validated and required by the database. Items will fail to be added/edited if a mandatory field is not defined and properly formatted.
Search Boolean False Add field to the list of fields to match search strings to.
index Boolean True Show field in table view as a column. False values will hide the column field from the table.
thumbnail Boolean False Use field as footer for images on thumbnail mode
hidden Boolean False Hide field from profile (detailed) view
additional Boolean False Show field inside the "Additional Information" section of the profile view
multiedit Boolean False Allow multiple edition of this field (bulk operation). True adds the field to the multiple edition form on the Group View
Available Field Types
type description
string This type is by default. If a field has no 'type' property, then it is a string field. Allow you to store simple strings.
integer Store integer numbers
float Store float numbers
date Stores a date in ISO format. A datepicker is attached to its input field to help setting its value.
facet Stored as an array of strings, this field is used as a facet. It will show up on the facet index and allow you to use it as a filter. Counts for all its possible values will be also displayed. This is where the solr magic comes in.
tuple Relates a facet field with another facet field. i.e. team members have different roles for each of their teams. By making a tuple field 'team:role', we get filters for team, role, and a one to one correlation of them.
img Field stores a URL that points to an image. If its the only img field in your schema, it will render as a big profile picture on the profile view and allow users to upload images from the browser. A minified version of it will also be displayed on the table.
email Contains one email address. Form validation for email will be applied, and it will be rendered as a clickable mailto: link.
skype Containes one skype username. It will be rendered as a clickable skype chat link.
multiline Containes a long string. Rendered as a text-area instead of as a single line text field
dropdown Containes a definite set of options to select from. Render as a select element, it allows you to define options with the 'options' property
Available properties for facet fields
key type description
specials Array Each facet is splitted in two areas, separated by a gray line. Define this property with an array of strings and each facet field that matches will be displayed below the line.
classifier Array Defines this field as the category to sort items on thumbnail view. The order of this array will determine the order of the sections on that view.
order Array Define the order of the facet fields. If not specified, fields will be sorted alphabetically.
collapse Boolean Define wich facets should show collapsed initially (folded) on the facet index. Once you click on them, the state is stored in localStorage, overriding this setting
Available properties for dropdown fields
key type description
options Array Defined as an array of strings for the availablo options in the dropdown menu.
